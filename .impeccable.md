# Pharo Agents — Design Context

## Target Audience
Technical founders and AI team leads at Sapira (a venture builder). They manage a fleet of AI agents running on OpenClaw and need visibility into agent health, usage, and configuration.

## Use Cases
- Monitor AI agent fleet health at a glance (which agents are active, how much they're consuming)
- Deep-dive into individual agent usage, configuration, and skills
- Manage agent skills and configuration from a web interface
- Track token consumption and context window pressure

## Brand Personality / Tone
- **Professional but modern** — enterprise SaaS, not consumer app
- **Dark mode primary** — feels like a control center / mission control
- **Sapira blue (#1e40af)** as primary brand color
- Clean, data-dense, no wasted space
- Feels like a premium observability tool (think: Vercel dashboard, Linear, Raycast)

## Aesthetic Direction
Industrial-minimal with precision — think mission control for AI agents. Dark backgrounds, subtle borders, data-forward cards, clean typography. Not flashy, not boring — quietly impressive. The kind of interface where every pixel has a reason to exist.

## Technical Constraints
- Next.js 15, App Router, TypeScript
- Tailwind CSS
- Must work on desktop and tablet (min 768px)
- Dark mode is the default and primary experience

## Differentiation
The context usage bars and token metrics should feel alive — like monitoring vital signs. Each agent card should feel like a character dossier, not just a data row.
